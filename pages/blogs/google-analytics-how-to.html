How to Add Google Analytics to Your GitHub Pages Website

Tracking your website visitors helps you understand traffic patterns, user engagement, and where your visitors are coming from. This guide walks you through the exact steps to set up Google Analytics (GA4) on a GitHub Pages (github.io) site, with clear explanations at every stage.

Step 1: Create a Google Analytics Property

1. Open Google Analytics

Go to https://analytics.google.com and sign in using your Google account. Make sure this is the account you want to use for managing analytics.

2. Open the Admin Panel

Once signed in, find and click the Admin button (a gear icon in the bottom left). This opens the settings area where you can manage accounts, properties, and tracking settings.

3. Create or Select an Analytics Account

If this is your first time using Google Analytics, click Create Account. Enter a name that makes sense to you (for example, your website name). If you already have an account, simply select it from the list — you do not need a new account if an existing one will work.

4. Create a GA4 Property

Within the selected account, click Create Property. A property represents the website or application you want to track. Enter a descriptive name (such as your site's name or URL), choose the correct time zone, and select your reporting currency. Continue through the setup prompts. Business details or analytics goals are optional and do not affect the basic tracking setup.

5. Add a Web Data Stream

After creating the property, Google Analytics will prompt you to add a Data Stream. Choose Web as the platform. Enter your GitHub Pages URL, and be sure to include the https:// prefix — for example: 

https://yourusername.github.io
Once you finish adding the Web data stream, Google Analytics will immediately show the full tracking code snippet (including your Measurement ID) directly on the page. You’ll use this snippet in the next step — there’s no need to copy just the ID separately.

Step 2: Add the Tracking Code to Your GitHub Pages Site

The tracking code (called gtag.js) is what allows Google Analytics to collect data about visits and user behavior. After completing the setup in the previous step, the analytics dashboard shows the code snippet along with your Measurement ID.

Your tracking snippet will look similar to this:

<script async src="https://www.googletagmanager.com/gtag/js?id=G-XXXXXXXXXX"></script>
<script>
  window.dataLayer = window.dataLayer || [];
  function gtag(){dataLayer.push(arguments);}
  gtag('js', new Date());
  gtag('config', 'G-XXXXXXXXXX');
</script>

To install it:

  • Open your GitHub repo in your editor or on GitHub.com
  • Find the HTML file(s) you want to track (for example, index.html)
  • Paste the tracking code inside the <head> section

Make sure the snippet appears before the closing </head> tag so it loads early when a visitor opens your page.

Step 3: Commit and Deploy

After inserting the tracking script:

  1. Save your changes
  2. Stage and commit them in Git
  3. Push the commit to your GitHub repository

GitHub Pages will automatically rebuild your site with the updated code. You don’t need to change any GitHub configuration to get analytics working.

Step 4: Verify Tracking

To confirm Google Analytics is collecting data:

  1. Open your website in a browser
  2. In Google Analytics, navigate to Reports > Realtime
  3. If tracking is working correctly, you should see at least one active user (yourself)

The realtime dashboard updates quickly, so you should see activity almost instantly.

Final Notes

  • Some data may take a few minutes to appear in the full reports
  • GA4 is the current version of Google Analytics supported by Google
  • You can also monitor site performance and traffic using the Google Analytics mobile app
